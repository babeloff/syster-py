= HirSymbol
:description: Reference for all HirSymbol attributes and kind values.
:toc: left
:toclevels: 2

[source,python]
----
class syster_py.HirSymbol
----

A semantic symbol extracted from a SysML v2 or KerML model element.
Returned by `SyntaxFile.symbols()` and `Database.file_symbols()`.

The name "HIR" stands for _High-level Intermediate Representation_ — the layer above the raw syntax tree where elements have been resolved to named symbols with qualified names and relationships.

Instances are frozen (all attributes are read-only) and are not constructible directly from Python.

'''

== Attributes

=== Identity

[cols="1,1,3",options="header"]
|===
| Attribute | Type | Description

| `name` | `str` | Simple (unqualified) name as it appears in source.
| `qualified_name` | `str` | Fully qualified name using `::` as separator, e.g. `VehicleModel::Wheel::rim`.
| `element_id` | `str` | UUID-based stable identifier assigned by the SysML/KerML standard to each element.
| `kind` | `str` | Display name of the element kind — see <<kind-values>> below.
|===

=== Location

All positions are *0-indexed*.

[cols="1,1,3",options="header"]
|===
| Attribute | Type | Description

| `file_id` | `int` | The integer ID passed to `symbols()` or managed by `Database`.
| `start_line` | `int` | 0-indexed line of the first character of the element declaration.
| `start_col` | `int` | 0-indexed column of the first character.
| `end_line` | `int` | 0-indexed line of the last character.
| `end_col` | `int` | 0-indexed column of the last character.
|===

=== Documentation

[cols="1,1,3",options="header"]
|===
| Attribute | Type | Description

| `doc`
| `str \| None`
| The content of the `doc /* ... */` comment attached to the element, or `None` if absent or not yet extracted by the current syster-base version.
|===

=== Relationships

[cols="1,1,3",options="header"]
|===
| Attribute | Type | Description

| `supertypes`
| `list[str]`
| Qualified names of all directly specialised types (`:>` in SysML, `specializes` in KerML).
  Empty list if no supertypes are declared.
|===

=== Modifiers

[cols="1,1,3",options="header"]
|===
| Attribute | Type | Description

| `is_public` | `bool` | `True` if the element is publicly visible (no `private` keyword).
| `is_abstract` | `bool` | `True` if the element has the `abstract` modifier.
| `is_variation` | `bool` | `True` if the element has the `variation` modifier (SysML v2 variation point).
| `is_readonly` | `bool` | `True` if the usage has the `readonly` modifier.
| `is_derived` | `bool` | `True` if the usage has the `derived` modifier.
|===

'''

== Known limitations

=== Missing fields: direction and multiplicity

`HirSymbol` does not expose two common feature properties:

* *Direction* — the `in` / `out` / `inout` / `return` keyword on action parameters.
* *Multiplicity* — the `[0..1]`, `[0..*]`, `[4]` etc. bracket expression on usages.

Both are present in the source text but not yet extracted by syster-base into the HIR.
The workaround is to read them from the source line via `SyntaxFile.source_text()`:

[source,python]
----
import re

def extract_direction(source: str, sym: syster_py.HirSymbol) -> str | None:
    """Return 'in', 'out', 'inout', or 'return' if present on the symbol's source line."""
    line = source.splitlines()[sym.start_line]
    m = re.search(r'\b(in|out|inout|return)\b', line)
    return m.group(1) if m else None

def extract_mult(source: str, sym: syster_py.HirSymbol) -> str | None:
    """Return the bracketed multiplicity string, e.g. '[0..*]', or None."""
    line = source.splitlines()[sym.start_line]
    m = re.search(r'\[([^\]]+)\]', line)
    return m.group(0) if m else None
----

These are upstream API gaps — direction and multiplicity should be filed as feature requests against syster-base.

=== Error-recovery placeholder names

When the parser cannot fully parse a declaration it still emits a synthetic symbol whose name contains angle brackets, for example `<anon#1@l103>` or `<:String#2@L28>`.
These are error-recovery artefacts and should not be treated as real model elements.
Filter them before using the symbol list:

[source,python]
----
def is_real_symbol(sym: syster_py.HirSymbol) -> bool:
    return '<' not in sym.name

real_syms = [s for s in sf.symbols(file_id=1) if is_real_symbol(s)]
----

This is a bug in syster-base — the placeholder names should not surface in the public `HirSymbol` list.

=== `#Annotation{ }` metadata blocks before `use case def`

Files that place inline `#TaskRef { ... }` (or similar `#Name { }` metadata) immediately before a `use case def` inside a package body cause syster-base 0.1.0 to fail to produce `"Use case def"` symbols for the entire file.
The file still parses (no `ParseError` is raised) but the use-case symbols are silently absent.

Detect affected files with a presence check:

[source,python]
----
import re

ANNOTATION_BEFORE_USECASE = re.compile(
    r'#\w+\s*\{[^}]*\}\s*use\s+case\s+def', re.DOTALL)

def has_annotation_usecase_bug(source: str) -> bool:
    return bool(ANNOTATION_BEFORE_USECASE.search(source))
----

This is a parser bug — `#Annotation{ }` blocks before `use case def` in a package body should be handled without suppressing the following symbol.

== Example — inspect a symbol

[source,python]
----
import syster_py

source = """
package Shapes {
    abstract part def Shape {
        doc /* Base shape type */
    }
    part def Circle :> Shape {
        attribute radius : Real;
    }
}
"""

sf = syster_py.parse_sysml(source)
for sym in sf.symbols(file_id=1):
    if sym.name == "Circle":
        print(f"name:          {sym.name}")
        print(f"qualified:     {sym.qualified_name}")
        print(f"kind:          {sym.kind}")
        print(f"supertypes:    {sym.supertypes}")
        print(f"is_abstract:   {sym.is_abstract}")
        print(f"location:      {sym.start_line}:{sym.start_col}")
----

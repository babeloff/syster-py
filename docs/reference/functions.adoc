= Module-level functions
:description: Reference for version(), parse_sysml(), and parse_kerml().

== `version`

[source,python]
----
syster_py.version() -> str
----

Returns the version string of the installed `syster-py` extension, as set in `Cargo.toml`.
Example: `"0.1.0"`.

'''

== `parse_sysml`

[source,python]
----
syster_py.parse_sysml(text: str) -> SyntaxFile
----

Parse `text` as a SysML v2 source string.

=== Parameters

[cols="1,1,3",options="header"]
|===
| Name | Type | Description

| `text` | `str` | The complete source text to parse.
|===

=== Returns

A link:syntax-file.adoc[`SyntaxFile`] whose `is_sysml()` method returns `True`.

=== Never raises

The underlying syster-base parser is error-recovering: it always produces a concrete syntax tree even when `text` contains mistakes.
Syntax problems are recorded as link:diagnostics.adoc[`ParseDiagnostic`] objects accessible via `SyntaxFile.errors()`.

=== Example

[source,python]
----
sf = syster_py.parse_sysml("package P { part def Wheel; }")
assert sf.is_sysml()
assert not sf.has_errors()
----

'''

== `parse_kerml`

[source,python]
----
syster_py.parse_kerml(text: str) -> SyntaxFile
----

Parse `text` as a KerML source string.

=== Parameters

[cols="1,1,3",options="header"]
|===
| Name | Type | Description

| `text` | `str` | The complete source text to parse.
|===

=== Returns

A link:syntax-file.adoc[`SyntaxFile`] whose `is_kerml()` method returns `True`.

=== Never raises

Same error-recovery semantics as `parse_sysml`.

=== Example

[source,python]
----
kf = syster_py.parse_kerml("package K { datatype Mass; }")
assert kf.is_kerml()
assert not kf.has_errors()
----
